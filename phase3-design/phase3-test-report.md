## 三、关键发现

### 3.1 正向发现

1. **困难场景改善明显**
   - 实验组在微信、反爬站点成功率100%
   - 对照组在困难场景有1个失败（90%）

2. **智能工具选择有效**
   - 实验组使用 `predicted_best` 策略
   - 对已知困难域名自动选择browser

### 3.2 问题发现

1. **Fallback次数增加**
   - 对照组平均0.43次，实验组1.17次
   - 实验组策略导致更多尝试

2. **边界场景适配不足**
   - JSON API等特殊格式场景
   - 实验组的通用策略不适合

## 四、优化建议

### 4.1 策略调整

| 调整 | 说明 |
|------|------|
| **添加格式检测** | 对JSON API使用web_fetch而非智能策略 |
| **优化Fallback顺序** | 对边界场景使用保守顺序 |
| **调整并行策略** | 仅对真正困难的域名启用并行 |

### 4.2 配置修改建议

```json
{
  "name": "treatment_optimized_v2",
  "format_detection": {
    "enabled": true,
    "json_urls": ["api.github.com", "*.json"],
    "tool_override": "web_fetch"
  },
  "parallel_fetch": {
    "enabled": true,
    "domains": ["weixin.qq.com", "mp.weixin.qq.com", "zhihu.com"],
    "exclude": ["api.*", "*.json"]
  }
}
```
